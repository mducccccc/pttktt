Câu 1 — Độ phức tạp thuật toán đệ quy
Cho code đệ quy → Bước 1: tìm biểu thức truy hồi T(n) · Bước 2: dùng cây hoặc Master Theorem
1Bước 1 — Tìm biểu thức truy hồi T(n)

Nhìn vào hàm đệ quy và đặt T(n) = thời gian chạy với đầu vào kích thước n.

  • Xác định base case: thường là T(0) = 1 hoặc T(1) = 1
  • Đếm số lần gọi đệ quy (= a)
  • Xác định kích thước mỗi lần gọi (= n/b hoặc n-k)
  • Đếm công việc ngoài đệ quy (= f(n))
  • Viết: T(n) = a·T(n/b) + f(n)
Ví dụ
C++
void foo(int n) {
    if (n <= 1) return;         // base case: T(1) = 1
    for(int i=0; i<n; i++) ...    // vòng lặp: f(n) = O(n)
    foo(n/2);                      // 1 lần gọi, kích thước n/2
    foo(n/2);                      // 1 lần gọi, kích thước n/2
}

→ T(n) = 2T(n/2) + n, T(1) = 1

2Bước 2A — Master Theorem

Áp dụng khi T(n) = aT(n/b) + f(n), với a ≥ 1, b > 1, f(n) > 0.

Tính p = logb(a), so sánh với f(n):

Trường hợp 1
f(n) = O(n^(p-ε))
f(n) nhỏ hơn n^p
T(n) = Θ(n^p)
Trường hợp 2
f(n) = Θ(n^p)
f(n) bằng n^p
T(n) = Θ(n^p · log n)
Trường hợp 3
f(n) = Ω(n^(p+ε))
f(n) lớn hơn n^p
T(n) = Θ(f(n))
Ví dụ 1 — T(n) = 2T(n/2) + n

a=2, b=2, f(n)=n → p = log₂(2) = 1

n^p = n¹ = n, mà f(n) = n → Trường hợp 2 → T(n) = Θ(n log n)

Ví dụ 2 — T(n) = 4T(n/2) + n

a=4, b=2, f(n)=n → p = log₂(4) = 2

n^p = n², mà f(n) = n = O(n^(2-1)) → Trường hợp 1 → T(n) = Θ(n²)

Ví dụ 3 — T(n) = T(n/2) + n²

a=1, b=2, f(n)=n² → p = log₂(1) = 0

n^p = n⁰ = 1, mà f(n) = n² = Ω(n^(0+2)) → Trường hợp 3 → T(n) = Θ(n²)

2Bước 2B — Cây đệ quy (Recursion Tree)
  • Vẽ cây: gốc là T(n), mỗi nút có a nhánh con kích thước n/b
  • Tại mỗi tầng i: chi phí = a^i · f(n/b^i)
  • Số tầng = logb(n), số lá = a^(logb(n)) = n^(logb(a))
  • Tổng = cộng chi phí tất cả tầng + chi phí lá
Ví dụ — T(n) = 2T(n/2) + n (MergeSort)

Tầng 0: 1 nút, chi phí = n

Tầng 1: 2 nút, chi phí = 2·(n/2) = n

Tầng 2: 4 nút, chi phí = 4·(n/4) = n

... log₂n tầng, mỗi tầng chi phí n → Tổng = n·log₂n = Θ(n log n)

⚠ Đặc biệt — T(n) = T(n-1) + c

Khi đệ quy giảm tuyến tính (n-1), không dùng Master Theorem. Dùng cây: n tầng, mỗi tầng c → T(n) = Θ(n)

BFS — Mọi con đường về 0
Dùng Stack + tập Output: phân tích u = a×b → sinh v = (a−1)×(b+1), tìm tất cả giá trị có thể đến từ n
Ý tưởng
Minh họa
Code C++

Bài toán: Từ n, dùng phép biến đổi u = a×b → v = (a−1)×(b+1). Tìm tất cả giá trị có thể sinh ra (kể cả 0).

  • B1: Nhập n
  • B2: Khai báo Stack S và tập Output
  • B3: Đưa n vào S và Output
  • B4: Lấy u = S.top() ra khỏi stack — duyệt mọi cách phân tích u = a×b (a ≤ √u):
    • Sinh v = (a−1)×(b+1)
    • Nếu v chưa có trong Output → đưa v vào S và Output
  • B5: Lặp lại B4 cho đến khi Stack rỗng
  • B6: Duyệt xuất tất cả phần tử trong Output
Tại sao a ≤ √u?
Mỗi cặp (a, b) với a×b = u và a ≤ b chỉ cần xét một lần — cặp (b, a) cho kết quả đối xứng đã được bao gồm
⚠ Ví dụ phân tích

u = 30 = 1×30 → v = 0×31 = 0  |  2×15 → 1×16 = 16  |  3×10 → 2×11 = 22  |  5×6 → 4×7 = 28

Ví dụ n = 12 — theo dõi Stack và Output từng bước
Bước Xét (u) Next (v sinh ra) Stack S Output
0 — — [12] {12}
1 12 0  7  10 [0, 7, 10] {12, 0, 7, 10}
2 10 0*  6 [0, 7, 6] {12, 0, 7, 10, 6}
3 6 0*  4 [0, 7, 4] {…, 10, 6, 4}
4 4 0*  3 [0, 7, 3] {…, 6, 4, 3}
5 3 0* [0, 7] {…, 6, 4, 3}
6 7 0* [0] {…, 6, 4, 3}
7 0 — [] {…, 6, 4, 3}
Kết quả Output (sắp xếp): {0, 3, 4, 6, 7, 10, 12}

* Giá trị màu xám = đã có trong Output, bỏ qua không đưa vào Stack.
Giá trị vàng = phần tử trên cùng Stack (sẽ được xét tiếp theo).

C++ — Ví dụ n = 6
#include <iostream>
#include <stack>
#include <set>
using namespace std;

int main() {
    int n = 6; // B1: n = 6

    stack<int> S;   // B2: khai báo Stack
    set<int> output; // B2: khai báo tập Output

    // B3: đưa n vào S và output
    S.push(n);
    output.insert(n);

    // B4 + B5: lặp cho đến khi stack rỗng
    while (!S.empty()) {
        int u = S.top(); S.pop(); // lấy u khỏi stack

        // duyệt phân tích u = a*b, a <= sqrt(u)
        for (int a = 1; a * a <= u; a++) {
            if (u % a == 0) {
                int b = u / a;
                int v = (a - 1) * (b + 1); // sinh v = (a-1)*(b+1)
                if (!output.count(v)) {   // v chưa có trong output
                    S.push(v);
                    output.insert(v);
                }
            }
        }
    }

    // B6: xuất tất cả phần tử output
    for (int x : output)
        cout << x << " ";
    cout << "\n";
    // Kết quả in ra: 0 3 4 6
}
Trace n = 6

Push 6 → Output={6}, Stack=[6]

Pop 6: a=1→v=0 (push), a=2→v=4 (push) → Stack=[0,4]

Pop 4: a=1→v=0 (bỏ*), a=2→v=3 (push) → Stack=[0,3]

Pop 3: a=1→v=0 (bỏ*) → Stack=[0]

Pop 0: a×a≤0 → không có v → Stack=[]

→ Output in ra: 0 3 4 6

Độ phức tạp: O(n × √n) — mỗi giá trị duyệt ≤ √n ước số

DFS/BFS — Rôbot (Biến đổi tọa độ)
Dùng Stack + tập Output để đếm số vị trí rôbot có thể đi được từ vị trí xuất phát
Ý tưởng
Minh họa
Code C++

Bài toán: Rôbot ở vị trí (x, y). Tại mỗi bước, rôbot có thể di chuyển theo quy tắc:

  • Đến (y, x/2) nếu x chẵn
  • Đến ((y+1)/2, x) nếu y lẻ

Cho vị trí xuất phát, đếm số vị trí rôbot có thể đi được.

Ý tưởng (dùng Stack duyệt các trạng thái):

  • B1: Nhập tọa độ xuất phát (x, y)
  • B2: Khai báo Stack S và tập Output (set) để lưu các vị trí đã duyệt
  • B3: Đưa (x, y) vào S và Output
  • B4: Lấy (u, v) khỏi S:
    • Nếu u chẵn: xét z = (v, u/2), nếu z chưa có trong Output → đưa z vào S và Output
    • Nếu v lẻ: xét z = ((v+1)/2, u), nếu z chưa có trong Output → đưa z vào S và Output
  • B5: Lặp lại B4 cho đến khi S rỗng
  • B6: In ra số lượng phần tử trong Output (đây là số vị trí đi được)
⚠ Lưu ý

Bài toán có thể giải bằng DFS (dùng Stack) hoặc BFS (dùng Queue). Tập Output (set hoặc mảng đánh dấu 2D) đóng vai trò đảm bảo không lặp vô hạn.

Ví dụ (x, y) = (2, 3) — theo dõi Stack và Output từng bước
Bước Xét (u, v) Next (z sinh ra) Stack S Output
0 — — [(2,3)] {(2,3)}
1 (2, 3) (3,1) (2,2) [(3,1), (2,2)] {(2,3), (3,1), (2,2)}
2 (2, 2) (2,1) [(3,1), (2,1)] {…, (2,1)}
3 (2, 1) (1,1) (1,2) [(3,1), (1,1), (1,2)] {…, (1,1), (1,2)}
4 (1, 2) — [(3,1), (1,1)] {…}
5 (1, 1) (1,1)* [(3,1)] {…}
6 (3, 1) (1,3) [(1,3)] {…, (1,3)}
7 (1, 3) (2,1)* [] {…}
Kết quả Output có 7 phần tử. Số vị trí đi được = 7

* Giá trị màu xám = đã có trong Output, bỏ qua không đưa vào Stack.
Giá trị vàng = phần tử trên cùng Stack (sẽ được xét tiếp theo).

C++ — Ví dụ (2, 3)
#include <iostream>
#include <stack>
#include <set>
using namespace std;

int main() {
    int sx = 2, sy = 3; // B1: Nhập vị trí xuất phát

    stack<pair<int, int>> S;   // B2: Khai báo Stack S
    set<pair<int, int>> output; // B2: Khai báo tập Output

    // B3: Đưa vị trí xuất phát vào S và output
    S.push({sx, sy});
    output.insert({sx, sy});

    // B4 + B5: Lặp cho đến khi Stack rỗng
    while (!S.empty()) {
        pair<int, int> curr = S.top();
        S.pop();
        int u = curr.first;
        int v = curr.second;

        // Nếu u chẵn: xét z = (v, u/2)
        if (u % 2 == 0) {
            pair<int, int> z = {v, u / 2};
            if (!output.count(z)) {
                S.push(z);
                output.insert(z);
            }
        }

        // Nếu v lẻ: xét z = ((v+1)/2, u)
        if (v % 2 != 0) {
            pair<int, int> z = {(v + 1) / 2, u};
            if (!output.count(z)) {
                S.push(z);
                output.insert(z);
            }
        }
    }

    // B6: In ra số lượng vị trí đi được
    cout << "So vi tri di duoc: " << output.size() << "\n";
    
    cout << "Cac vi tri do la:\n";
    for (auto p : output) {
        cout << "(" << p.first << ", " << p.second << ") ";
    }
    cout << "\n";
    
    return 0;
}
BFS — Đong nước
Bài toán 2 bình: tìm số bước tối thiểu để đong được đúng c lít
Ý tưởng
Minh họa
Code C++

Bài toán: Có 2 bình dung tích A và B lít. Tìm số thao tác tối thiểu để được đúng c lít (trong 1 trong 2 bình). Nguồn nước vô hạn.

Các thao tác có thể:

  • Đổ đầy bình A (hoặc B)
  • Đổ rỗng bình A (hoặc B)
  • Đổ từ A sang B (hoặc B sang A)

Ý tưởng (BFS trên không gian trạng thái):

  • Trạng thái = (lượng nước trong A, lượng nước trong B) = (x, y)
  • Trạng thái ban đầu: (0, 0)
  • BFS từ (0,0), mỗi trạng thái sinh ra ≤ 6 trạng thái kề
  • Khi x=c hoặc y=c → đã đạt mục tiêu
Ví dụ A=4, B=3, c=2

Bước 0: (0,0)

Bước 1: Đổ đầy A → (4,0)

Bước 2: Đổ A sang B → (1,3)

Bước 3: Đổ rỗng B → (1,0)

Bước 4: Đổ A sang B → (0,1)

Bước 5: Đổ đầy A → (4,1)

Bước 6: Đổ A sang B → (2,3)

→ Bình A có 2 lít, đạt mục tiêu sau 6 bước

C++
#include <iostream>
#include <vector>
#include <queue>
#include <map>
using namespace std;
typedef pair<int,int> pii;

int bfsWater(int A, int B, int c) {
    map<pii, int> dist;
    queue<pii> q;
    q.push(make_pair(0,0)); dist[make_pair(0,0)] = 0;
    while (!q.empty()) {
        int x = q.front().first, y = q.front().second; q.pop();
        int d = dist[make_pair(x,y)];
        if (x==c || y==c) return d;
        vector<pii> nexts = {
            make_pair(A, y), make_pair(x, B),
            make_pair(0, y), make_pair(x, 0),
            make_pair(x-min(x,B-y), y+min(x,B-y)),
            make_pair(x+min(y,A-x), y-min(y,A-x))
        };
        for (auto s : nexts)
            if (!dist.count(s)) { dist[s] = d+1; q.push(s); }
    }
    return -1;
}
Quay lui — Quân hậu (N-Queens)
Đặt N quân hậu trên bàn cờ N×N sao cho không quân nào ăn quân nào
Ý tưởng
🌳 Cây quay lui
Code C++

Ý tưởng: Đặt lần lượt từng hàng, thử từng cột trong hàng đó. Nếu ô (row, col) an toàn thì đặt hậu, gọi đệ quy cho hàng tiếp theo. Nếu không an toàn thì thử cột khác (quay lui).

  • Hàm isSafe(row, col): kiểm tra cột, đường chéo trái, đường chéo phải
  • Hàm solve(row): thử đặt hậu vào từng cột của hàng row
  • Nếu row = N → tìm được 1 lời giải, in ra
  • Ngược lại: thử col từ 0 đến N-1, nếu safe → đặt → gọi solve(row+1) → bỏ
Kiểm tra an toàn
Không có 2 hậu cùng cột: col[i] ≠ col[row]
Không có 2 hậu cùng đường chéo: |row-i| ≠ |col[row]-col[i]|
🌳 Cây quay lui — N-Queens (N=4)
Nhấn Bắt đầu
Nhấn Bắt đầu để xem cây quay lui được xây dựng từng bước.
📖 Chú giải màu

■ Xanh dương = đang xét  |  ■ Xanh lá = đặt thành công  |  ■ Đỏ = bị cắt nhánh (không an toàn)

C++
#include <iostream>
using namespace std;

int n, col[100];

bool isSafe(int row, int c) {
    for (int i = 0; i < row; i++)
        if (col[i] == c || abs(row-i) == abs(col[i]-c))
            return false;
    return true;
}

void solve(int row) {
    if (row == n) {
        for (int i=0; i<n; i++) cout << col[i]+1 << " ";
        cout << "\n";
        return;
    }
    for (int c = 0; c < n; c++) {
        if (isSafe(row, c)) {
            col[row] = c;
            solve(row + 1);
        }
    }
}

int main() {
    cin >> n;
    solve(0);
}

Quay lui — Chia của
Chia n món đồ thành 2 nhóm có tổng giá trị bằng nhau
Ý tưởng
🌳 Cây quay lui
Code C++

Bài toán: Cho n món đồ có giá trị v[1..n]. Chia thành 2 nhóm sao cho tổng 2 nhóm bằng nhau.

Điều kiện cần: Tổng S phải là số chẵn. Bài toán quy về: tìm tập con có tổng = S/2.

  • Tính tổng S, kiểm tra S % 2 == 0, đặt target = S/2
  • Hàm solve(i, sum): xét món thứ i với tổng nhóm 1 hiện tại là sum
  • Thử cho vào nhóm 1: solve(i+1, sum + v[i])
  • Thử cho vào nhóm 2: solve(i+1, sum)
  • Khi i = n và sum = target → tìm được cách chia
🌳 Cây quay lui — Chia của: v=[3,1,4], target=4
Nhấn Bắt đầu
Nhấn Bắt đầu để xem cây quay lui được xây dựng từng bước.
v=[3,1,4], S=8, target=4. Mỗi nút hiện thị (i, sum).
📖 Chú giải màu

■ Xanh dương = đang xét  |  ■ Xanh lá = tìm được cách chia  |  ■ Đỏ = cắt nhánh (sum > target)

C++
#include <iostream>
using namespace std;

int n, v[100], group[100], target;
bool found = false;

void solve(int i, int sum) {
    if (found) return;
    if (i == n) {
        if (sum == target) { found = true; /* in kết quả */ }
        return;
    }
    if (sum > target) return; // cắt nhánh
    group[i] = 1; solve(i+1, sum + v[i]);
    group[i] = 2; solve(i+1, sum);
}

int main() {
    cin >> n;
    int S = 0;
    for(int i=0;i<n;i++) { cin>>v[i]; S+=v[i]; }
    if (S % 2 != 0) { cout << "Khong the chia!\n"; return 0; }
    target = S / 2;
    solve(0, 0);
}
Quay lui — Đổi tiền
Liệt kê tất cả các cách đổi tiền dùng quay lui
Ý tưởng
Minh họa
Code C++

Bài toán: Có các mệnh giá d[1..k] (vô hạn mỗi loại). Liệt kê TẤT CẢ các cách đổi số tiền n.

  • Hàm solve(idx, remain, path): đang xét mệnh giá thứ idx, còn lại remain, đường đi path
  • Để tránh trùng: mỗi bước chỉ dùng mệnh giá từ idx trở đi (dùng ≥ idx)
  • Thử dùng d[idx] k lần (k = 0,1,2,...) miễn remain - k*d[idx] ≥ 0
  • Gọi đệ quy với idx+1, remain đã giảm
  • Khi remain = 0 → in đường đi
Ví dụ: d = [1, 2, 5], n = 5

Các cách đổi 5:

1+1+1+1+1 = 5

1+1+1+2 = 5

1+2+2 = 5

5 = 5

→ Tổng 4 cách: {1×5}, {1×3, 2×1}, {1×1, 2×2}, {5×1}

C++
#include <iostream>
using namespace std;

int d[] = {1, 2, 5}, k = 3, n;
int cnt[10];

void solve(int idx, int remain) {
    if (remain == 0) {
        for (int i=0; i<k; i++)
            if (cnt[i]>0) cout << d[i] << "x" << cnt[i] << " ";
        cout << "\n"; return;
    }
    if (idx == k) return;
    for (int c = remain/d[idx]; c >= 0; c--) {
        cnt[idx] = c;
        solve(idx+1, remain - c*d[idx]);
    }
    cnt[idx] = 0;
}

int main() {
    cin >> n; solve(0, n);
}
Tham lam — Lập lịch (Activity Selection)
Chọn nhiều công việc nhất trong khoảng thời gian không chồng chéo
Ý tưởng
▶ Minh họa từng bước
Code C++

Bài toán: Có n công việc, công việc i có thời gian bắt đầu s[i] và kết thúc f[i]. Chọn nhiều công việc nhất sao cho không có 2 công việc nào chồng thời gian.

Chiến lược tham lam: Luôn chọn công việc kết thúc sớm nhất chưa chồng với công việc đã chọn.

  • Sắp xếp các công việc theo thời gian kết thúc tăng dần
  • Chọn công việc đầu tiên (kết thúc sớm nhất)
  • Duyệt qua các công việc còn lại: nếu s[i] ≥ thời điểm kết thúc của công việc cuối đã chọn → chọn
  • Cập nhật thời điểm kết thúc cuối
Điều kiện chọn
s[i] ≥ lastFinish → chọn công việc i, cập nhật lastFinish = f[i]
▶ Thuật toán lập lịch tham lam — từng bước
Nhấn Bắt đầu
Nhấn Bắt đầu để xem thuật toán tham lam từng bước.
C++
#include <iostream>
#include <algorithm>
using namespace std;

struct Job { int s, f, id; };

int main() {
    int n; cin >> n;
    vector<Job> jobs(n);
    for(auto& j : jobs) cin >> j.s >> j.f;
    for(int i=0;i<n;i++) jobs[i].id = i+1;
    
    sort(jobs.begin(), jobs.end(), [](Job a, Job b){
        return a.f < b.f;
    });
    
    int lastFinish = -1, count = 0;
    for(auto& j : jobs) {
        if(j.s >= lastFinish) {
            cout << "Chon job " << j.id << "\n";
            lastFinish = j.f; count++;
        }
    }
    cout << "Tong so: " << count << "\n";
}

Tham lam — Khiêu vũ
Ghép cặp nam-nữ để tổng chênh lệch chiều cao nhỏ nhất
Ý tưởng
▶ Minh họa từng bước

Bài toán: Có n nam (chiều cao m[]) và n nữ (chiều cao f[]). Ghép mỗi nam với một nữ. Tối thiểu hóa tổng |m[i] - f[σ(i)]|.

Chiến lược tham lam: Sắp xếp cả 2 mảng, ghép người có rank tương đương với nhau.

  • Sắp xếp m[] tăng dần
  • Sắp xếp f[] tăng dần
  • Ghép m[i] với f[i] (cùng thứ hạng)

Hàng đợi ưu tiên — Nối thanh kim loại
Nối các thanh với chi phí nhỏ nhất (Huffman-like)
Ý tưởng
Minh họa
Code C++

Bài toán: Có n thanh kim loại độ dài a[1..n]. Mỗi lần nối 2 thanh có chi phí = tổng độ dài 2 thanh đó. Tối thiểu hóa tổng chi phí nối.

Chiến lược (Tham lam + Min-Heap): Luôn nối 2 thanh ngắn nhất trước.

  • Đưa tất cả độ dài vào min-heap (hàng đợi ưu tiên min)
  • Lặp n-1 lần: lấy 2 phần tử nhỏ nhất a và b
  • Chi phí += a + b
  • Đưa (a+b) trở lại heap
  • Khi heap còn 1 phần tử → xong
Tại sao luôn nối ngắn nhất?
Thanh nào nối sớm hơn sẽ được cộng nhiều lần hơn → nối ngắn trước giảm chi phí
Ví dụ: [4, 3, 2, 6]

Heap: {2, 3, 4, 6}

Bước 1: lấy 2+3=5, chi phí=5. Heap: {4, 5, 6}

Bước 2: lấy 4+5=9, chi phí=5+9=14. Heap: {6, 9}

Bước 3: lấy 6+9=15, chi phí=14+15=29. Heap: {15}

→ Tổng chi phí tối thiểu = 29

C++
#include <iostream>
#include <queue>
using namespace std;

int main() {
    int n; cin >> n;
    priority_queue<long long, vector<long long>, greater<long long>> pq;
    for(int i=0;i<n;i++) { long long x; cin>>x; pq.push(x); }
    long long cost = 0;
    while(pq.size() > 1) {
        long long a = pq.top(); pq.pop();
        long long b = pq.top(); pq.pop();
        cost += a + b; pq.push(a + b);
    }
    cout << cost << "\n";
}

Độ phức tạp: O(n log n)

Hàng đợi ưu tiên — Trinh thám (Dijkstra)
Tìm đường đi ngắn nhất trong đồ thị có trọng số
Ý tưởng
Minh họa
Code C++

Bài toán: Đồ thị có trọng số không âm. Tìm đường đi ngắn nhất từ nguồn s đến tất cả đỉnh còn lại.

Thuật toán Dijkstra:

  • Khởi tạo dist[s]=0, dist[v]=∞ với v≠s
  • Đưa (0, s) vào min-priority queue
  • Lấy đỉnh u có dist nhỏ nhất ra khỏi queue
  • Với mỗi cạnh (u,v,w): nếu dist[u]+w < dist[v] → cập nhật dist[v], đưa vào queue
  • Lặp đến khi queue rỗng
⚠ Điều kiện

Dijkstra chỉ đúng khi trọng số cạnh ≥ 0. Nếu có cạnh âm → dùng Bellman-Ford.

Ví dụ đồ thị 4 đỉnh, nguồn = 1

Cạnh: 1-2 (w=2), 1-3 (w=6), 2-3 (w=3), 2-4 (w=8), 3-4 (w=1)

Ban đầu: dist=[0, ∞, ∞, ∞]

Lấy 1 (d=0): dist[2]=2, dist[3]=6

Lấy 2 (d=2): dist[3]=5, dist[4]=10

Lấy 3 (d=5): dist[4]=6

→ dist = [0, 2, 5, 6]

C++
#include <iostream>
#include <queue>
#include <vector>
using namespace std;
typedef pair<int,int> pii;
const int INF = 1e9;

vector<int> dijkstra(int n, vector<vector<pii>>& adj, int s) {
    vector<int> dist(n+1, INF);
    priority_queue<pii, vector<pii>, greater<pii>> pq;
    dist[s] = 0; pq.push(make_pair(0, s));
    while(!pq.empty()) {
        int d = pq.top().first, u = pq.top().second; pq.pop();
        if(d > dist[u]) continue;
        for(int i=0; i<(int)adj[u].size(); i++) {
            int v=adj[u][i].first, w=adj[u][i].second;
            if(dist[u]+w < dist[v]) {
                dist[v] = dist[u]+w; pq.push(make_pair(dist[v], v));
            }
        }
    }
    return dist;
}

Độ phức tạp: O((V+E) log V)

Hàng đợi ưu tiên — Phần tử trung vị
Tìm trung vị của dòng số liệu sử dụng 2 heap
Ý tưởng
Minh họa
Code C++

Bài toán: Nhận liên tục các số, sau mỗi số in ra trung vị của tất cả số đã nhận.

Ý tưởng — 2 Heap:

  • maxHeap (L): chứa nửa nhỏ hơn, đỉnh = max của nửa nhỏ
  • minHeap (R): chứa nửa lớn hơn, đỉnh = min của nửa lớn
  • Đưa x vào L (maxHeap)
  • Nếu L.top() > R.top() → cân bằng: chuyển L.top() sang R
  • Cân bằng kích thước: nếu |L|-|R| > 1 → chuyển L.top() sang R; hoặc ngược lại
  • Trung vị = L.top() nếu L nhiều hơn, hoặc (L.top()+R.top())/2
Ví dụ: nhập lần lượt 5, 1, 3, 7, 2

x=5: L={5}, R={}. Median=5

x=1: L={1}, R={5}. Median=(1+5)/2=3

x=3: L={3,1}, R={5}. Median=3

x=7: L={3,1}, R={5,7}. Median=(3+5)/2=4

x=2: L={3,2,1}, R={5,7}. Median=3

C++
#include <iostream>
#include <queue>
using namespace std;

priority_queue<int> L;
priority_queue<int, vector<int>, greater<int>> R;

void addNum(int x) {
    L.push(x);
    if (!R.empty() && L.top() > R.top()) { R.push(L.top()); L.pop(); }
    if (L.size() > R.size() + 1) { R.push(L.top()); L.pop(); }
    else if (R.size() > L.size()) { L.push(R.top()); R.pop(); }
}

double getMedian() {
    if (L.size() > R.size()) return L.top();
    return (L.top() + R.top()) / 2.0;
}

Độ phức tạp: O(log n) mỗi phần tử

Hàng đợi ưu tiên — Giao hàng
Tối ưu hóa lịch giao hàng sử dụng hàng đợi ưu tiên
Ý tưởng
Minh họa
Code C++

Bài toán: Có n đơn hàng, đơn i có thời điểm đến a[i] và thời gian xử lý p[i]. Tối thiểu hóa tổng thời gian chờ (Shortest Job First).

  • Sắp xếp đơn hàng theo thời điểm đến a[i]
  • Dùng min-heap chứa (p[i], a[i]) — ưu tiên đơn hàng xử lý ngắn
  • Tại thời điểm t: đưa tất cả đơn đến trước t vào heap
  • Lấy đơn có p[i] nhỏ nhất ra xử lý, cập nhật t
Chiến lược SJF
Trong số các đơn đã đến, xử lý đơn ngắn nhất trước → tối thiểu tổng thời gian chờ
Ví dụ: 3 đơn hàng
Đơn Đến (a) Xử lý (p)
1 0 6
2 1 2
3 2 4

t=0→6 (đơn 1), t=6→8 (đơn 2), t=8→12 (đơn 3)

Tổng chờ: (6-0)+(8-1)+(12-2) = 23

C++
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;
typedef pair<int,int> pii;

int main() {
    int n; cin >> n;
    vector<pii> orders(n);
    for(auto& o:orders) cin>>o.first>>o.second;
    sort(orders.begin(),orders.end());
    priority_queue<pii, vector<pii>, greater<pii>> pq;
    long long t=0, total=0; int idx=0;
    while(idx<n || !pq.empty()) {
        while(idx<n && orders[idx].first<=t)
            pq.push(make_pair(orders[idx].second, orders[idx].first)), idx++;
        if(pq.empty()) { t=orders[idx].first; continue; }
        int p=pq.top().first, a=pq.top().second; pq.pop();
        t += p; total += t - a;
    }
    cout << total << "\n";
}
Quy hoạch động — Đổi tiền
Tìm số đồng xu ít nhất để đổi thành n đồng
Ý tưởng
📊 Bảng QHĐ động
Code C++

Bài toán: Có các mệnh giá d[1..k] (vô hạn mỗi loại). Tìm số đồng xu ÍT NHẤT để tổng = n.

dp[i] = min số xu để đổi i đồng
dp[0] = 0
dp[i] = min(dp[i - d[j]] + 1) với mọi d[j] ≤ i
📊 Điền bảng QHĐ — đổi tiền: d=[1,3,4], n=6
Nhấn Bắt đầu
Nhấn Bắt đầu để xem bảng QHĐ được điền từng ô.
C++
#include <iostream>
#include <algorithm>
using namespace std;
const int INF = 1e9;

int main() {
    int k, n; cin >> k >> n;
    vector<int> d(k);
    for(auto& x:d) cin>>x;
    vector<int> dp(n+1, INF);
    dp[0] = 0;
    for(int i=1; i<=n; i++)
        for(int j=0; j<k; j++)
            if(i >= d[j] && dp[i-d[j]] != INF)
                dp[i] = min(dp[i], dp[i-d[j]] + 1);
    cout << dp[n] << "\n";
}

Quy hoạch động — Xâu con chung dài nhất (LCS)
Tìm xâu con chung dài nhất của 2 xâu X và Y
Ý tưởng
📊 Bảng QHĐ động
Code C++
dp[i][j] = độ dài LCS của X[1..i] và Y[1..j]

Nếu X[i] == Y[j]: dp[i][j] = dp[i-1][j-1] + 1
Ngược lại: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
📊 Điền bảng LCS — X="ABCB", Y="BDCAB"
Nhấn Bắt đầu
Nhấn Bắt đầu để xem bảng LCS được điền từng ô.
C++
#include <iostream>
using namespace std;

int main() {
    string X, Y; cin >> X >> Y;
    int m=X.size(), n=Y.size();
    vector<vector<int>> dp(m+1, vector<int>(n+1, 0));
    for(int i=1; i<=m; i++)
        for(int j=1; j<=n; j++) {
            if(X[i-1] == Y[j-1]) dp[i][j] = dp[i-1][j-1] + 1;
            else dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
        }
    cout << dp[m][n] << "\n";
}

Quy hoạch động — Sắp xếp ba lô (0/1 Knapsack)
Chọn đồ vật có giá trị cao nhất mà không vượt quá trọng lượng
Ý tưởng
📊 Bảng QHĐ động
Code C++
dp[i][j] = giá trị lớn nhất chọn từ i vật đầu, trọng lượng ≤ j

Không lấy vật i: dp[i][j] = dp[i-1][j]
Lấy vật i (j ≥ w[i]): dp[i][j] = dp[i-1][j-w[i]] + v[i]
dp[i][j] = max(hai trường hợp)
📊 Điền bảng Ba lô — n=3, W=5, (w,v):(2,3),(3,4),(4,5)
Nhấn Bắt đầu
Nhấn Bắt đầu để xem bảng ba lô được điền từng ô.
C++
#include <iostream>
using namespace std;

int main() {
    int n, W; cin >> n >> W;
    vector<int> w(n+1), v(n+1);
    for(int i=1;i<=n;i++) cin>>w[i]>>v[i];
    vector<vector<int>> dp(n+1, vector<int>(W+1, 0));
    for(int i=1; i<=n; i++)
        for(int j=0; j<=W; j++) {
            dp[i][j] = dp[i-1][j];
            if(j >= w[i]) dp[i][j] = max(dp[i][j], dp[i-1][j-w[i]] + v[i]);
        }
    cout << dp[n][W] << "\n";
}

Độ phức tạp: O(n×W)

Quy hoạch động — Cắt hình vuông ít nhất
Cắt hình chữ nhật m×n thành ít hình vuông nhất
Ý tưởng
Minh họa
Code C++
dp[m][n] = số hình vuông ít nhất để cắt hình chữ nhật m×n

Base case: dp[k][k] = 1

Cắt ngang tại hàng i: dp[m][n] = min(dp[i][n] + dp[m-i][n])
Cắt dọc tại cột j: dp[m][n] = min(dp[m][j] + dp[m][n-j])
Ví dụ: m=2, n=3

dp[2][3]: thử cắt ngang i=1: dp[1][3]+dp[1][3]=6

thử cắt dọc j=1: dp[2][1]+dp[2][2]=2+1=3

thử cắt dọc j=2: dp[2][2]+dp[2][1]=1+2=3

→ dp[2][3] = 3

C++
#include <iostream>
using namespace std;
int dp[50][50];

int minSquares(int m, int n) {
    if (m == n) return 1;
    if (dp[m][n]) return dp[m][n];
    int res = m * n;
    for(int i=1; i<m; i++)
        res = min(res, minSquares(i,n) + minSquares(m-i,n));
    for(int j=1; j<n; j++)
        res = min(res, minSquares(m,j) + minSquares(m,n-j));
    return dp[m][n] = res;
}

Độ phức tạp: O(m²×n + m×n²)
