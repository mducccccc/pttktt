Câu 1 — Độ phức tạp thuật toán đệ quy
Cho code đệ quy → Bước 1: tìm biểu thức truy hồi T(n) · Bước 2: dùng cây hoặc Master Theorem
1Bước 1 — Tìm biểu thức truy hồi T(n)

Nhìn vào hàm đệ quy và đặt T(n) = thời gian chạy với đầu vào kích thước n.

  • Xác định base case: thường là T(0) = 1 hoặc T(1) = 1
  • Đếm số lần gọi đệ quy (= a)
  • Xác định kích thước mỗi lần gọi (= n/b hoặc n-k)
  • Đếm công việc ngoài đệ quy (= f(n))
  • Viết: T(n) = a·T(n/b) + f(n)
Ví dụ
C++
void foo(int n) {
    if (n <= 1) return;         // base case: T(1) = 1
    for(int i=0; i<n; i++) ...    // vòng lặp: f(n) = O(n)
    foo(n/2);                      // 1 lần gọi, kích thước n/2
    foo(n/2);                      // 1 lần gọi, kích thước n/2
}

→ T(n) = 2T(n/2) + n, T(1) = 1

2Bước 2A — Master Theorem

Áp dụng khi T(n) = aT(n/b) + f(n), với a ≥ 1, b > 1, f(n) > 0.

Tính p = logb(a), so sánh với f(n):

Trường hợp 1
f(n) = O(n^(p-ε))
f(n) nhỏ hơn n^p
T(n) = Θ(n^p)
Trường hợp 2
f(n) = Θ(n^p)
f(n) bằng n^p
T(n) = Θ(n^p · log n)
Trường hợp 3
f(n) = Ω(n^(p+ε))
f(n) lớn hơn n^p
T(n) = Θ(f(n))
Ví dụ 1 — T(n) = 2T(n/2) + n

a=2, b=2, f(n)=n → p = log₂(2) = 1

n^p = n¹ = n, mà f(n) = n → Trường hợp 2 → T(n) = Θ(n log n)

Ví dụ 2 — T(n) = 4T(n/2) + n

a=4, b=2, f(n)=n → p = log₂(4) = 2

n^p = n², mà f(n) = n = O(n^(2-1)) → Trường hợp 1 → T(n) = Θ(n²)

Ví dụ 3 — T(n) = T(n/2) + n²

a=1, b=2, f(n)=n² → p = log₂(1) = 0

n^p = n⁰ = 1, mà f(n) = n² = Ω(n^(0+2)) → Trường hợp 3 → T(n) = Θ(n²)

2Bước 2B — Cây đệ quy (Recursion Tree)
  • Vẽ cây: gốc là T(n), mỗi nút có a nhánh con kích thước n/b
  • Tại mỗi tầng i: chi phí = a^i · f(n/b^i)
  • Số tầng = logb(n), số lá = a^(logb(n)) = n^(logb(a))
  • Tổng = cộng chi phí tất cả tầng + chi phí lá
Ví dụ — T(n) = 2T(n/2) + n (MergeSort)

Tầng 0: 1 nút, chi phí = n

Tầng 1: 2 nút, chi phí = 2·(n/2) = n

Tầng 2: 4 nút, chi phí = 4·(n/4) = n

... log₂n tầng, mỗi tầng chi phí n → Tổng = n·log₂n = Θ(n log n)

⚠ Đặc biệt — T(n) = T(n-1) + c

Khi đệ quy giảm tuyến tính (n-1), không dùng Master Theorem. Dùng cây: n tầng, mỗi tầng c → T(n) = Θ(n)

BFS — Mọi con đường về 0
Dùng Stack + tập Output: phân tích u = a×b → sinh v = (a−1)×(b+1), tìm tất cả giá trị có thể đến từ n
Ý tưởng
Minh họa
Code C++

Bài toán: Từ n, dùng phép biến đổi u = a×b → v = (a−1)×(b+1). Tìm tất cả giá trị có thể sinh ra (kể cả 0).

  • B1: Nhập n
  • B2: Khai báo Stack S và tập Output
  • B3: Đưa n vào S và Output
  • B4: Lấy u = S.top() ra khỏi stack — duyệt mọi cách phân tích u = a×b (a ≤ √u):
    • Sinh v = (a−1)×(b+1)
    • Nếu v chưa có trong Output → đưa v vào S và Output
  • B5: Lặp lại B4 cho đến khi Stack rỗng
  • B6: Duyệt xuất tất cả phần tử trong Output
Tại sao a ≤ √u?
Mỗi cặp (a, b) với a×b = u và a ≤ b chỉ cần xét một lần — cặp (b, a) cho kết quả đối xứng đã được bao gồm
⚠ Ví dụ phân tích

u = 30 = 1×30 → v = 0×31 = 0  |  2×15 → 1×16 = 16  |  3×10 → 2×11 = 22  |  5×6 → 4×7 = 28

Ví dụ n = 12 — theo dõi Stack và Output từng bước
Bước Xét (u) Next (v sinh ra) Stack S Output
0 — — [12] {12}
1 12 0  7  10 [0, 7, 10] {12, 0, 7, 10}
2 10 0*  6 [0, 7, 6] {12, 0, 7, 10, 6}
3 6 0*  4 [0, 7, 4] {…, 10, 6, 4}
4 4 0*  3 [0, 7, 3] {…, 6, 4, 3}
5 3 0* [0, 7] {…, 6, 4, 3}
6 7 0* [0] {…, 6, 4, 3}
7 0 — [] {…, 6, 4, 3}
Kết quả Output (sắp xếp): {0, 3, 4, 6, 7, 10, 12}

* Giá trị màu xám = đã có trong Output, bỏ qua không đưa vào Stack.
Giá trị vàng = phần tử trên cùng Stack (sẽ được xét tiếp theo).

C++ — Ví dụ n = 6
#include <iostream>
#include <stack>
#include <set>
using namespace std;

int main() {
    int n = 6; // B1: n = 6

    stack<int> S;   // B2: khai báo Stack
    set<int> output; // B2: khai báo tập Output

    // B3: đưa n vào S và output
    S.push(n);
    output.insert(n);

    // B4 + B5: lặp cho đến khi stack rỗng
    while (!S.empty()) {
        int u = S.top(); S.pop(); // lấy u khỏi stack

        // duyệt phân tích u = a*b, a <= sqrt(u)
        for (int a = 1; a * a <= u; a++) {
            if (u % a == 0) {
                int b = u / a;
                int v = (a - 1) * (b + 1); // sinh v = (a-1)*(b+1)
                if (!output.count(v)) {   // v chưa có trong output
                    S.push(v);
                    output.insert(v);
                }
            }
        }
    }

    // B6: xuất tất cả phần tử output
    for (int x : output)
        cout << x << " ";
    cout << "\n";
    // Kết quả in ra: 0 3 4 6
}
Trace n = 6

Push 6 → Output={6}, Stack=[6]

Pop 6: a=1→v=0 (push), a=2→v=4 (push) → Stack=[0,4]

Pop 4: a=1→v=0 (bỏ*), a=2→v=3 (push) → Stack=[0,3]

Pop 3: a=1→v=0 (bỏ*) → Stack=[0]

Pop 0: a×a≤0 → không có v → Stack=[]

→ Output in ra: 0 3 4 6

Độ phức tạp: O(n × √n) — mỗi giá trị duyệt ≤ √n ước số

Cho 2 can n lít, m lít. Tìm số bước đong ít nhất để được k lít
Ý tưởng
Minh họa
Code C++

Bài toán: Có 2 can dung tích n và m lít. Tìm số thao tác đong nước ít nhất để được đúng k lít (trong 1 trong 2 can). Nguồn nước vô hạn.

Ý tưởng:

  • B1: Khai báo Queue, Map
  • B2: nhập n,m,k
  • B3: Đưa (0,0) vào Q
  • B4: M[(0,0)]=1
  • B5: lấy (x,y) ra khỏi Q đặt z=x+y
    Next[] = {(0,y); (n,y); (x,0); (x,m); (max(0,z-m), min(z,m)); (min(z,n), max(0,z-n))}
    Duyệt v trên next
    Nếu v khác M thì đưa v vào Q, M[v] = M[(x,y)] + 1
    Nếu v first == k hoặc v second == k thì return M[v] - 1
  • B6: lặp lại B5 đến khi Q rỗng
  • B7: return -1
Ví dụ n = 3, m = 5, k = 4 — theo dõi Queue và Map từng bước

* Giá trị — = tất cả trạng thái sinh ra đều đã tồn tại trong Map, không đẩy thêm vào Queue.
* Giá trị vàng = phần tử ở đầu Queue (sẽ lấy ra xét ở bước tiếp theo).

C++
//Tichpx - Dong nuoc
#include<bits/stdc++.h>
using namespace std;

int main()
{
    int n,m,k;
    cin>>n>>m>>k;
    queue<pair<int,int>>Q;
    map<pair<int,int>,int> M;
    Q.push({0,0});
    M[{0,0}]=0;
    while(Q.size())
    {
        int x=Q.front().first, y=Q.front().second, z=x+y;
        Q.pop();
        pair<int,int> Next[]={{0,y},{n,y},{x,0},{x,m},{max(0,z-m),min(z,m)},{min(z,n),max(0,z-n)}};
        for(auto v:Next)
            if(M.find(v)==M.end())
            {
                Q.push(v);
                M[v]=M[{x,y}]+1;
                if(v.first==k or v.second==k) return !printf("%d",M[v]); 
            }
    }
    printf("khong dong duoc");
}
Quay lui — Chia của
Chia n món đồ cho 2 người sao cho chênh lệch tổng giá trị là nhỏ nhất
Ý tưởng
🌳 Cây quay lui
Code C++

Bài toán: Cho n món đồ có giá trị a[0...n-1]. Chia thành 2 người sao cho chênh lệch min.

Ý tưởng:

  • B1: Nhập n, mảng a[0...n-1]. Khởi tạo res = ∞, T = a[0] + ... + a[n-1]
  • B2: Hàm đệ quy TRY(k, A, B) giả sử đã chia được k đồ vật (từ a[0]...a[k-1]). Tổng người 1 là A, tổng người 2 là B
  • TH1: k = n → Đã chia xong n đồ vật. Cập nhật res = B - A.
  • TH2: k < n → Đang chia đồ vật thứ k (giá trị a[k]):
    • Nếu A + a[k] ≤ T / 2 thì đệ quy TRY(k+1, A+a[k], B) (Đưa cho người 1).
    • Nếu B + a[k] < (T + res) / 2 thì đệ quy TRY(k+1, A, B+a[k]) (Đưa cho người 2).
⚠ Lưu ý

Biến res lưu trữ chênh lệch nhỏ nhất tìm được. Các điều kiện A + a[k] ≤ T/2 và B + a[k] < (T+res)/2 dùng để cắt nhánh (Pruning), loại bỏ nhanh chóng các trường hợp chắc chắn không cho kết quả tốt hơn.

📖 Chú giải màu

■ Xanh dương = đang xét  |  ■ Xanh lá = duyệt đến đích (cập nhật res)  |  ■ Đỏ = bị cắt nhánh (điều kiện sai)

C++
//Tichpx - chia cua
#include<bits/stdc++.h>
using namespace std;

int n, a[100], T=0, res=1e9;

void TRY(int k, int A, int B)
{
    if(k == n) { 
        res = B - A; 
        return; 
    }
    if(A + a[k] <= T / 2) 
        TRY(k + 1, A + a[k], B);
        
    if(B + a[k] < (T + res) / 2) 
        TRY(k + 1, A, B + a[k]);
}

int main()
{
    cin >> n;
    for(int i = 0; i < n; i++) {
        cin >> a[i]; 
        T += a[i];
    }
    TRY(0, 0, 0);
    cout << res;
}
Quay lui — Đổi tiền
Tìm số tờ tiền ít nhất để đổi số tiền M
Ý tưởng
🌳 Cây quay lui
Code C++

Bài toán: Cho n mệnh giá tiền a[1]...a[n] và số tiền M. Hãy tìm số tờ tiền ít nhất để đổi được M.

Ý tưởng chính
  • Sử dụng thuật toán Quay lui (Backtracking) để duyệt qua các số lượng tờ có thể dùng cho mỗi mệnh giá.
  • Trạng thái: TRY(k, t, T)
    • k: Đang xét mệnh giá a[k]
    • t: Tổng số tờ đã dùng trước đó
    • T: Tổng số tiền đã đổi được trước đó

Luồng chạy thuật toán:

  • B1: Khởi tạo kỉ lục res = ∞
  • B2 (Trường hợp dừng): k = n (Đã đến mệnh giá cuối cùng)
    • Nếu phần tiền còn lại (M - T) chia hết cho mệnh giá a[n] thì ta dùng thêm (M - T) / a[n] tờ.
    • Cập nhật kỉ lục: res = min(res, t + (M - T) / a[n])
  • B3 (Đệ quy): k < n
    • Gọi z là số tờ sẽ dùng cho mệnh giá a[k].
    • Điều kiện duyệt (Nhánh cận): Duyệt z từ 0 trở đi với điều kiện:
      z + t < res (Tờ ≤ kỉ lục) T + z * a[k] ≤ M (Tiền ≤ M)
    • Gọi đệ quy: TRY(k+1, t+z, T+z*a[k])

📖 Chú giải màu

■ Xanh dương = đang xét  |  ■ Xanh lá = đến đích (cập nhật res)  |  ■ Đỏ = cắt nhánh (pruning)

C++
//Tichpx
#include<bits/stdc++.h>
using namespace std;

int a[1000], n, M, res = 1e9;

void TRY(int k, int t, int T) // t: so to da doi, T: tong tien da doi
{
    if(k == n)
    {
        if((M - T) % a[n] == 0) 
            res = min(res, t + (M - T) / a[n]);
    }
    else 
    {
        for(int z = 0; z + t < res and T + z * a[k] <= M; z++) 
            TRY(k + 1, t + z, T + z * a[k]);
    }
}

int main()
{
    cin >> n >> M;
    for(int i = 1; i <= n; i++) 
        cin >> a[i];
        
    TRY(1, 0, 0);
    
    if(res == 1e9) 
        cout << "khong doi duoc";
    else 
        cout << res;
}
Tham lam — Lập lịch
Chọn nhiều việc nhất mà không chồng chéo kể cả mút
Ý tưởng
▶ Minh họa từng bước
Code C++

Bài toán: Cho n việc, mỗi việc có thời điểm bắt đầu a_i và kết thúc b_i. Chọn nhiều việc nhất sao cho không có 2 việc nào chồng chéo (kể cả thời điểm chạm mút chung).

Ý tưởng thuật toán
  • B1: Nhập vào danh sách các việc gồm <a1, b1>...<an, bn>.
  • B2: Sắp xếp tăng dần theo thời điểm kết thúc (b). Vì ta muốn kết thúc càng sớm càng tốt để nhường thời gian cho việc khác.
  • B3: Khởi tạo số lượng việc đã chọn res = 0 và thời điểm kết thúc của việc cuối cùng được chọn là t = -∞.
  • B4: Duyệt i = 0 đến n - 1:
    • Nếu thời điểm bắt đầu của việc hiện tại lớn hơn thời điểm kết thúc t (tức là a_i > t), ta sẽ chọn việc này.
    • Tăng số lượng việc: res++.
    • Cập nhật thời điểm kết thúc mới: t = b_i.
  • B5: Xuất kết quả res.
▶ Lập lịch: a = [4, 2, 4, 3, 16, 13, 10], b = [7, 8, 8, 5, 22, 15, 17]
Nhấn Bắt đầu
Nhấn Bắt đầu để xem thuật toán tham lam từng bước.
📖 Chú giải màu

■ Xanh dương = đang xét  |  ■ Xanh lá = chọn (thoả mãn a > t)  |  ■ Đỏ = loại bỏ (chồng chéo a ≤ t)

C++
//Tichpx
#include<bits/stdc++.h>
using namespace std;

typedef pair<int, int> pii;
#define kt first
#define bd second

int main()
{
    int n, res = 0, t = -INT_MAX;
    cin >> n;
    pii A[n];
    for(auto &a : A) cin >> a.bd >> a.kt;
    
    sort(A, A + n);
    
    for(auto a : A) {
        if(a.bd > t) {
            res++; 
            t = a.kt;
        }
    }
    cout << res;
}
Hàng đợi ưu tiên — Trinh thám
Tìm giá trị lớn nhất trong mỗi cửa sổ trượt kích thước k
Ý tưởng
Minh họa
Code C++

Bài toán: Cho n số nguyên a[1...n] và số k. Cứ mỗi k phần tử liên tiếp, hãy tìm phần tử lớn nhất.

Ý tưởng thuật toán
  • B1: Khai báo HĐUT Q (first: giá trị, second: vị trí).
  • B2: Nhập n và k.
  • B3: Duyệt i = 1 → n.
  • B4: Nhập a[i], đưa (a[i], i) vào Q.
  • B5: Nếu i ≥ k: Lặp xóa phần tử đỉnh nếu i - Q.top().second ≥ k.
  • B6: Xuất Q.top().first.

Hàng đợi ưu tiên — Phần tử trung vị
Tìm trung vị của dòng số liệu sử dụng 2 heap
Ý tưởng
Minh họa
Code C++

Bài toán: Trung vị của dãy a1, ..., an là phần tử đứng ở vị trí (n+1)/2 sau khi sắp xếp tăng dần. Cho dãy a1, ..., an, hãy tìm lần lượt trung vị của các dãy con {a1}, {a1, a2}, ..., {a1, ..., an}.

Ý tưởng thuật toán (Sử dụng 2 Heap)
  • B1: Khai báo Hàng đợi ưu tiên lớn (Max-Heap) L (chứa nửa bé).
  • B2: Khai báo Hàng đợi ưu tiên bé (Min-Heap) R (chứa nửa lớn).
  • B3: Nhập số lượng phần tử n.
  • B4: Duyệt i = 1 → n:
    • Nhập a[i].
    • Nếu i lẻ: L.push(a[i]).
    • Ngược lại (i chẵn): R.push(a[i]).
    • Nếu i > 1 và L.top() > R.top(): Đảo 2 phần tử đỉnh cho nhau (Lấy top của L đưa vào R và ngược lại).
    • Xuất L.top() chính là trung vị của dãy hiện tại.
Minh họa với mảng: 7 4 2 3 6 1 8 9 5
Bước i a[i] L (Max-Heap) R (Min-Heap) L.top > R.top? Trung vị
1 7 {7} {} - 7
2 4 {4} {7} 7 > 4 (Có) 4
3 2 {4, 2} {7} 4 > 7 (Không) 4
4 3 {3, 2} {7, 4} 4 > 3 (Có) 3
5 6 {4, 3, 2} {7, 6} 6 > 4 (Có) 4
6 1 {3, 2, 1} {7, 6, 4} 4 > 1 (Có) 3
7 8 {4, 3, 2, 1} {7, 6, 8} 8 > 4 (Có) 4
8 9 {4, 3, 2, 1} {7, 6, 8, 9} 4 > 6 (Không) 4
9 5 {5, 4, 3, 2, 1} {7, 6, 8, 9} 5 > 6 (Không) 5

→ Kết quả: 7 4 4 3 4 3 4 4 5

C++
//Tichpx - phan tu trung vi
#include<bits/stdc++.h>
using namespace std;

int main()
{
    priority_queue<int> L;   
    priority_queue<int, vector<int>, greater<int>> R;
    int n, x;
    cin >> n;
    for(int i = 1; i <= n; i++)
    {
        cin >> x;
        i % 2 ? L.push(x) : R.push(x);
        if(i > 1 && L.top() > R.top())
        {
            R.push(L.top());
            L.push(R.top());
            L.pop();
            R.pop();
        }
        cout << L.top() << " ";
    }
}

Hàng đợi ưu tiên — Giao hàng
Bài toán giao hàng có hạn chót (deadline) và tiền thưởng
Ý tưởng
Minh họa
Code C++

Bài toán: Có n món hàng, mỗi món có hạn chót ai và tiền thưởng bi. Mỗi đơn vị thời gian chỉ giao được 1 món. Món hàng i chỉ có thể giao tại thời điểm t ≤ ai. Tìm tổng tiền thưởng lớn nhất.

  • B1: Khai báo mảng vector A[100005] để nhóm các phần thưởng theo hạn chót.
  • B2: Khai báo hàng đợi ưu tiên lớn (Max-Heap) Q để lưu các phần thưởng đang "khả dụng".
  • B3: Nhập số lượng món hàng n.
  • B4: Duyệt i = 1 → n: Nhập ai, bi và đưa vào A[ai].push_back(bi).
  • B5: Khởi tạo res = 0.
  • B6: Duyệt ngược thời gian i = 105 → 1:
    • Đưa tất cả phần thưởng trong A[i] vào Q.
    • Nếu Q khác rỗng: res += Q.top(); Q.pop();
Chiến lược tham lam
Tại mỗi thời điểm t (xét từ muộn về sớm), ta chọn món có tiền thưởng cao nhất trong số các món có hạn chót ≥ t chưa được giao.
Ví dụ minh họa

Cho a = [3, 3, 1, 2, 2, 6] và b = [5, 7, 3, 4, 2, 1]

Thời gian i Món mới (A[i]) Q (Max-Heap) Chọn res
6 {1} {1} 1 1
5 — {} — 1
4 — {} — 1
3 {5, 7} {7, 5} 7 8
2 {4, 2} {5, 4, 2} 5 13
1 {3} {4, 3, 2} 4 17

→ Tổng tiền thưởng lớn nhất: 17

C++ — Tichpx
//Tichpx - giao hang
#include<bits/stdc++.h>
using namespace std;

int main()
{
    long res = 0, n, a, b;
    vector<int> A[100005];  // Các vector tạo ra mặc định rỗng
    priority_queue<int> Q;
    cin >> n;
    while(n--)
    {
        cin >> a >> b;
        A[a].push_back(b);
    }
    for(int i = 1e5; i > 0; i--)
    {
        for(auto x : A[i]) Q.push(x);
        if(Q.size()) 
        {
            res += Q.top();
            Q.pop();
        }
    }
    cout << res;
}
Quy hoạch động — Đổi tiền
Tìm số đồng xu ít nhất để đổi thành m đồng (Sử dụng bảng 2 chiều)
Ý tưởng
📊 Bảng QHĐ động
Code C++

Bài toán: Có n mệnh giá tiền a[1...n]. Tìm số tờ ít nhất để có tổng bằng m.

  • B1: Nhập n, m và mảng các mệnh giá a[1...n].
  • B2: Khai báo ma trận C[n+1][m+1]. Trong đó C[i][j] là số tờ ít nhất để đổi được số tiền j dùng các mệnh giá từ a[1...i].
  • B3: Khởi tạo cột đầu tiên: C[i][0] = 0 (để đổi 0 đồng cần 0 tờ).
  • B4: Khởi tạo hàng đầu tiên: C[0][j] = ∞ (không có mệnh giá nào thì không đổi được tiền > 0).
  • B5: Duyệt i = 1 → n và j = 1 → m:
    • Nếu a[i] > j: C[i][j] = C[i-1][j] (mệnh giá i quá lớn, lấy kết quả của bước trước).
    • Ngược lại: C[i][j] = min(C[i-1][j], 1 + C[i][j - a[i]]) (chọn tối ưu giữa việc không dùng và có dùng mệnh giá i).
  • B6: Kết quả là C[n][m].
📊 Điền bảng ma trận QHĐ — đổi tiền: a={5,1,7,3}, m=18
Nhấn Bắt đầu
Nhấn Bắt đầu để xem bảng QHĐ được điền từng ô.
C++ — Tichpx
//Tichpx - Doi tien QHD
#include<bits/stdc++.h>
using namespace std;

int n, m, a[100], C[100][1000] = {};

void TRACE(int n, int m) {
    if (C[n][m] == 0) return;
    while (C[n][m] == C[n - 1][m]) n--;
    TRACE(n, m - a[n]);
    cout << a[n] << " ";
}

int main() {
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> a[i];
    for (int j = 1; j <= m; j++) C[0][j] = 1e9;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (a[i] > j) C[i][j] = C[i - 1][j];
            else C[i][j] = min(C[i - 1][j], 1 + C[i][j - a[i]]);
    
    if (C[n][m] == 1e9) cout << "khong doi duoc";
    else {
        cout << "\nSo to " << C[n][m] << "\n";
        TRACE(n, m);
    }
    return 0;
}
Quy hoạch động — Xâu con chung dài nhất (LCS)
Tìm xâu con chung dài nhất của 2 xâu X và Y
Ý tưởng
📊 Bảng QHĐ động
Code C++
dp[i][j] = độ dài LCS của X[1..i] và Y[1..j]

Nếu X[i] == Y[j]: dp[i][j] = dp[i-1][j-1] + 1
Ngược lại: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
📊 Điền bảng LCS — X="ABCB", Y="BDCAB"
Nhấn Bắt đầu
Nhấn Bắt đầu để xem bảng LCS được điền từng ô.
C++
#include <iostream>
using namespace std;

int main() {
    string X, Y; cin >> X >> Y;
    int m=X.size(), n=Y.size();
    vector<vector<int>> dp(m+1, vector<int>(n+1, 0));
    for(int i=1; i<=m; i++)
        for(int j=1; j<=n; j++) {
            if(X[i-1] == Y[j-1]) dp[i][j] = dp[i-1][j-1] + 1;
            else dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
        }
    cout << dp[m][n] << "\n";
}

Độ phức tạp: O(m×n)

Quy hoạch động — Sắp xếp ba lô (0/1 Knapsack)
Chọn đồ vật có giá trị cao nhất mà không vượt quá trọng lượng
Ý tưởng
📊 Bảng QHĐ động
Code C++
dp[i][j] = giá trị lớn nhất chọn từ i vật đầu, trọng lượng ≤ j

Không lấy vật i: dp[i][j] = dp[i-1][j]
Lấy vật i (j ≥ w[i]): dp[i][j] = dp[i-1][j-w[i]] + v[i]
dp[i][j] = max(hai trường hợp)
📊 Điền bảng Ba lô — n=3, W=5, (w,v):(2,3),(3,4),(4,5)
Nhấn Bắt đầu
Nhấn Bắt đầu để xem bảng ba lô được điền từng ô.
C++
#include <iostream>
using namespace std;

int main() {
    int n, W; cin >> n >> W;
    vector<int> w(n+1), v(n+1);
    for(int i=1;i<=n;i++) cin>>w[i]>>v[i];
    vector<vector<int>> dp(n+1, vector<int>(W+1, 0));
    for(int i=1; i<=n; i++)
        for(int j=0; j<=W; j++) {
            dp[i][j] = dp[i-1][j];
            if(j >= w[i]) dp[i][j] = max(dp[i][j], dp[i-1][j-w[i]] + v[i]);
        }
    cout << dp[n][W] << "\n";
}

Độ phức tạp: O(n×W)

Quy hoạch động — Cắt hình vuông ít nhất
Cắt hình chữ nhật m×n thành ít hình vuông nhất
Ý tưởng
Minh họa
Code C++
dp[m][n] = số hình vuông ít nhất để cắt hình chữ nhật m×n

Base case: dp[k][k] = 1

Cắt ngang tại hàng i: dp[m][n] = min(dp[i][n] + dp[m-i][n])
Cắt dọc tại cột j: dp[m][n] = min(dp[m][j] + dp[m][n-j])
Ví dụ: m=2, n=3

dp[2][3]: thử cắt ngang i=1: dp[1][3]+dp[1][3]=6

thử cắt dọc j=1: dp[2][1]+dp[2][2]=2+1=3

thử cắt dọc j=2: dp[2][2]+dp[2][1]=1+2=3

→ dp[2][3] = 3

C++
#include <iostream>
using namespace std;
int dp[50][50];

int minSquares(int m, int n) {
    if (m == n) return 1;
    if (dp[m][n]) return dp[m][n];
    int res = m * n;
    for(int i=1; i<m; i++)
        res = min(res, minSquares(i,n) + minSquares(m-i,n));
    for(int j=1; j<n; j++)
        res = min(res, minSquares(m,j) + minSquares(m,n-j));
    return dp[m][n] = res;
}

Độ phức tạp: O(m²×n + m×n²)
